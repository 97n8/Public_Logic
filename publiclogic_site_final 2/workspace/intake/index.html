What this form does

This intake is meant to stay simple. It collects enough detail to decide what the work is and where it belongs. SharePoint and the flows handle the rest.

SharePoint list

In Microsoft 365 create a list such as PL_UniversalIntake with columns that match these field names:

  • OrgNameSingle line of text
  • PrimaryContactSingle line of text
  • ContactEmailSingle line of text
  • ContactPhoneSingle line of text
  • TimelineChoice
  • WorkTypeChoice
  • ModulesSingle line or multi choice
  • SummaryMultiple lines
  • NotesMultiple lines
  • SourceSingle line of text
  • StatusChoice

Flow outline

  • Trigger: When an HTTP request is received (Power Automate).
  • Parse the JSON body using these field names.
  • Create item in PL_UniversalIntake.
  • Use that item to build folders and files in SharePoint.
  • Post the project to the Processing board and send any alerts.

The form does not hold any data. Everything lives in SharePoint and in the flows.
