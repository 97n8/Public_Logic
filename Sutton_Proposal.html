Plain Talk (No buzzwords.)

If a resident can't submit a request without calling, if staff can't see what's due when, and if we can't export history on demand—it isn't better connections. SuttonCONNECT fixes those three things first, then connects everything else: resident intake → staff workflows → compliance exports → grant reporting → next year's funding. Massachusetts rules are baked in (Public Records 10/25 days, OML 48h posting, dog licensing renewals, retention schedules) with immutable audit logs as a staff liability shield.

ResidentsClear intake • Clear status Simple forms, email receipts, self-serve status page.
StaffQueues • Timers • Checklists Work shows up where it belongs. Deadlines are visible.
TownAudit-ready exports Routine JSON/PDF/CSV + checksums to the Town’s Drive.

Why Sutton Right Now

The Problem Without This

  • Institutional knowledge is in heads, inboxes, and desk drawers
  • New staff “learn by osmosis,” not by system
  • Phones become the workflow because status is opaque
  • Compliance is manual (timers in calendars, sticky notes)

The Change With VAULT + CLERK + FIX

  • Queue-driven work with visible clocks and escalation
  • SOPs + checklists embedded in the same place work happens
  • Resident status pages reduce “any update?” calls
  • Exports on a schedule with checksums → audit-ready by default
Succession resilience: when staff rotate, the system keeps Sutton’s know-how and compliance timers intact.

VAULT • CLERK • FIX (What they do)

VAULT (Agnostic Framework)

  • Universal naming (e.g., SUTTON_CLERK_PUBLIC_RECORD_2025-10-01_001_V1)
  • Case schema + audit trail + retention metadata
  • Compliance logic: PRR timers, OML postings, retention/export jobs
  • Town-owned configs and documentation (no lock-in)

CLERK (Administrative Backbone)

  • Public Records Requests (PRR)
  • Vital records requests (copies), licensing (dog, business, raffle)
  • Agenda posting & board/committee management
  • Elections intake, certifications, attestations, oaths

FIX (DPW & Facilities)

  • Potholes, signage, drainage, sidewalks, streetlights
  • Facility maintenance tickets, work orders, completions
  • Before/after photo guardrails (“no photo, no close”)
  • Exported histories feed grants and capital planning

Unified Operating Model (Resident → Staff → Exports)

1) Intake

  • Web forms, email gateway, phone-to-ticket (“person bridge”), or kiosk
  • Auto-name, classify, and route to the right queue
  • Email receipt with case # and expected timeline

2) Work

  • Checklists and SOPs in-line with the case
  • Timers: PRR 10/25 days, OML 48h, licensing renewals
  • Internal notes + immutable audit log

3) Export

  • Quarterly jobs → Google Shared Drive (JSON + PDF + CSV)
  • Manifest with SHA-256 checksums
  • On-demand per-case export (attachments included)

Concrete Scenarios (Week 1 changes)

Public Records Request (PRR)

  1. Resident submits PRR → clock starts automatically
  2. Countdown shows in Clerk queue: 10 business days to acknowledge
  3. If extension needed, template response with reasons + new date
  4. Exemption/redaction tracking + counsel review notes
  5. Export full request history on close (PDF+JSON+attachments)
Liability shield: immutable audit log records diligence even if a deadline slips.

Agenda Posting (Open Meeting Law)

  1. Board submits agenda → system validates required fields
  2. Posting reminders fire at 72/52/48 hours pre-meeting
  3. Timestamped “proof of posting” receipt
  4. Public status page shows latest version and time of posting

Dog License (New/Renewal)

  • Upload rabies cert → OCR validates dates, vet, tag #
  • Fee rules (spayed/neutered, senior discount, late fee)
  • License PDF + tag mailer generated with VAULT naming
  • Renewal reminders 30/15/3 days before expiration

Pothole Report (DPW)

  • Resident photo + map pin auto-captured
  • Supervisor assigns crew; SLA clock visible
  • “No close without after photo” enforced
  • Completion package (photos, materials, hours) → export

Compliance in the Flow of Work (not after)

Clocks & Escalations

  • PRR 10-day ack / 25-day max with extensions
  • OML 48-hour posting reminders w/ proof receipts
  • License renewal windows with late-fee logic
  • Escalation ladder: Clerk → Dept Head → TA

Audit Trail

  • Immutable history of actions and timestamps
  • Internal notes and counsel reviews retained
  • Per-case export includes index + attachments

Retention

  • Per-type retention rules in VAULT metadata
  • Quarterly export job + checksum manifest
  • Drive structure by category/year/sequence

AI Tiers (Augment, don’t replace)

Base (Included)
  • Auto-categorize + route cases
  • Urgent language detection
  • Receipt emails with plain-language summary
  • Suggest similar past cases to staff
Plus (Optional)
  • Draft responses for PRR (acknowledge/extend/fulfill)
  • Agenda → draft minutes (staff edits)
  • Seasonal workload predictions (snow, paving)
Pro (Optional)
  • Sutton-specific policy recall in drafts
  • Grant-ready narratives from exported metrics
  • “What-if” resource scenarios for budgets
Privacy: AI runs on town-approved providers; no training on resident PII; exports remain in the town’s Drive.

Risks & Liability (how we mitigate)

Adoption lag

  • Extend hypercare; desk-side coaching
  • Phone/paper bridges feed the same queues
  • Tune notifications & dashboards from usage

Compliance slip

  • Timers, reminders, & escalations pre-breach
  • Immutable audit log proves diligence
  • Counsel templates for extensions & exemptions

Platform issue

  • Data continues exporting to Drive
  • Configs documented as portable
  • 12-month transition support pledge
