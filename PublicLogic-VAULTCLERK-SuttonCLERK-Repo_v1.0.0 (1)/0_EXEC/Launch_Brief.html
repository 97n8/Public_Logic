Welcome to the VAULTCLERK™ Sutton Pilot — our first living deployment. The system is pre-wired for Google Workspace, Polimorphic, and PublicLogic™ compliance logic, with PublicInsight™ and CSS aligned for staff training and onboarding.

Open Executive Dashboard Download Full Pilot Package
Start Here
  • Open the Executive Dashboard — one-screen control center outlining the Promise, 90-Day Plan, Value, Risk, Stack, KPIs, Compliance, Training, and Checklist.
  • Unzip the package into a Shared Drive or local folder; the framework runs offline from the same directory.
  • Run Quick Setup (GWS) to apply Gmail labels, create a Drive watch-folder, and connect the 311 Sheet.
  • Use Smoke Test to confirm readiness, then open Binder Demo for an export preview.
What’s Inside
  • Framework Core: 21-folder VAULT doctrine (schema, manifests, SOPs, runtime healthchecks)
  • Governance & Spec: Pilot Spec and Circular for Town and Polimorphic reference
  • Quick Setup (GWS): Human-readable guide and JSON labels
  • Training Layer: PublicInsight™ (Allie) — readiness assessment and clerk micro-labs
  • Sample Data & Binder Demo: Three intakes with compiled binder preview
  • Smoke Test: One-click compliance readiness check
  • Integrity & Licensing: LICENSE, README, SHA256SUMS
Kickoff Goals
  • Week 1: Complete Quick Setup, Smoke Test, and Binder Demo; schedule PublicInsight™ micro-labs.
  • Week 2: Begin live intakes via /intake to validate schema and retention chain.
  • By Day 30: Unified queue, verified schema, binder export, and audit-ready compliance log.
  • CSS finalizes support cadence and grant alignment strategy.

Let’s launch. This becomes our baseline reference build once Sutton is live.
