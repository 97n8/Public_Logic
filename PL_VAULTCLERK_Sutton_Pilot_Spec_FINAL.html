VAULTCLERK — Sutton

Pilot Specification — Sutton (Polimorphic + Google Workspace)

Purpose. Establish a single, defensible municipal intake and records logic for all channels, with one queue, one schema, and defense‑ready exports. Town retains ownership of operational records and exports.

Stack: Polimorphic (data + automation), Google Workspace (Gmail, Drive, Sheets, Calendar), GIS, Email, Payments, Permitting.

1) Scope & Deliverables

  • Universal municipal intake form (all channels) mapped to a single schema.
  • VAULTCLERK logic: ID minting, timers, holds, retention tags, evidence links, export.
  • Ingress wiring for Gmail (RAO/Clerk), Drive watch‑folder scans, Sheets phone/311 log, POLI forms.
  • Clerk dashboard, binder builder (PDF/A), public copy workflow, and KPI reporting.
  • VAULT universal 21‑folder packaging with manifest and seed assets.
  • Staff SOP + micro‑labs; PublicInsight readiness and adoption support.

3) Universal Form & Schema

Universal Form (all channels)

  • One schema for web, counter, mail, email, phone/311, kiosk, referrals, state feeds, backfile, and API.
  • Channel‑specific details captured as metadata extensions, not forked forms.

Core Schema (excerpt)

  • record_id — CLERK‑YYYY‑#####
  • source_channel — web|counter|mail|email|phone|kiosk|referral|state|backfile|api
  • received_at — ISO 8601
  • retention_code — schedule key
  • deadline_at — policy‑derived
  • status — open|in_review|hold|closed|published
  • evidence_links[] — URIs to proofs (.pdfa, .eml, imgs)

4) Technical Architecture

Layers

  • Ingress — POLI, Gmail (RAO/Clerk), Drive scans, Sheets logs, referrals, state feeds, backfile, API.
  • Normalizer — OCR→classify→dedupe→metadata map to schema.
  • VAULTCLERK — ID minting, timers, holds, retention tags, hash, evidence links.
  • Outputs — Queue, Binder, Public copy, Integrations, Storage/Retention.

Interfaces (examples)

Google Workspace Wiring

  • Gmail shared inboxes (e.g., rao@…, clerk-intake@…) with labels vault/intake, vault/hold, vault/published.
  • Drive Shared Drive: Sutton_VAULTCLERK with watch‑folder /intake_scans.
  • Sheets phone/311 log (datetime,caller,subject,location,notes) synced to queue.
  • Scopes: Gmail/Drive/Sheets read‑only for ingestion; least privilege.

Security

  • Role‑based access; dual‑control for deletion.
  • Append‑only audit for state and export.
  • PDF/A normalization; redaction prior to public copy.

6) Implementation Plan

  1. Discovery & Wiring. Confirm Gmail/Drive/Sheets access, create shared inbox labels, set Drive watch‑folder, connect POLI forms.
  2. Schema & Normalizer. Lock universal schema; apply OCR/classify/dedupe rules.
  3. VAULTCLERK Logic. ID minting, timers, holds, retention tags, evidence links, audit.
  4. Outputs. Clerk dashboard, binder export, public copy workflow, integrations.
  5. Training. Staff micro‑labs and SOP walkthrough; PublicInsight readiness.
  6. Validation. Parallel run with sample intakes; acceptance against KPIs.
