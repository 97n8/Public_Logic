Summary

Public Logic and Capital Strategic Solutions (CSS) propose a municipal-first partnership to launch a Capital Improvement Plan (CIP) program with a CSS client town. CSS leads delivery and rollout. Public Logic supplies templates, process standards, and meeting-ready materials. The town keeps ownership and control of its plan and data.

This is a draft for discussion. It is meant to help a town quickly see what the program looks like, how decisions are made, and what it takes to adopt and maintain a CIP.

```

Partnership Model (Like Sutton/Polimorphic—Made for CSS)

This is a polimorphic model — a reusable framework that travels town to town. Just like the Sutton project, where templates and playbooks were refined and then deployed elsewhere, this CIP partnership creates standards that CSS can roll out repeatedly.

  • CSS leads implementation: run intake, workshops, scoring, affordability review, board packet prep, and public communications.
  • Public Logic supplies standards: plain-language templates, process playbooks, calendars, and checklists that travel town to town.
  • Town owns outcomes: final plan, data, schedules, and adoption decisions stay with the municipality.
  • Shared learning: CSS and Public Logic reuse anonymized lessons, templates, and metrics to improve future rollouts — a win-win-win.

What the CIP Module Does

  • Delivers a rolling 5–10 year plan with an adoptable Year 1 capital budget.
  • Defines "what counts as capital," sets thresholds, and standardizes project sheets.
  • Ranks projects with a transparent rubric so boards and residents can see the "why."
  • Shows affordability: debt capacity, pay-go, grants, and operating impacts.
  • Produces public-facing summaries for meetings and the website.

Process (Simple and Repeatable)

Annual Cycle

  • Fall: project intake and updates
  • Winter: scoring, draft plan, affordability checks
  • Spring: public meetings and adoption
  • Summer: grant work and project tracking

Meeting-Ready Materials

  • Capital policy statement (draft)
  • Project one-pagers (scope, cost, funding, O&M)
  • Ranked list with scoring notes
  • Slide deck for public hearing
  • Website summary and FAQ

CriterionWhat We Look For
Health & SafetyKnown hazards, code issues, risk of failure
Legal/RegulatoryMandates, time-bound orders or agreements
Asset ConditionInspection results, useful life, deferred maintenance
Service ImpactMaintains core services or expands capacity
External FundingGrants, reimbursements, partner dollars
ReadinessScope clarity, design/permitting status
Operating ImpactStaffing and O&M costs identified early

The rubric is built with department heads and finance so everyone agrees on the rules before ranking.
```
