PublicLogic × Polimorphic — VAULTCLERK Sutton Pilot Specification

Scope: Universal municipal intake & records logic using Polimorphic and Google Workspace. Includes PublicInsight™ training led by Dr. Allison Weiss Rothschild (Allie).

Ingress Channels

  • POLI forms → /intake
  • Gmail RAO/Clerk shared inboxes (labels vault/intake, vault/hold, vault/published)
  • Drive watch‑folder /intake_scans (300 dpi PDFs)
  • Phone/311 log (Google Sheets)

Core Logic

  • ID mint, timers, legal holds, retention tags
  • Evidence links; append‑only audit
  • Binder export (PDF/A + ZIP), public copy (redacted)

Interfaces (examples)

/intake            POST   Create from any channel
/records/{id}      GET    Record + history
/records/{id}/status PATCH State update (audited)
/export/binder     GET    Annual binder (PDF/A + ZIP)
/public/{id}       GET    Publishable copy

Security

  • Least‑privilege roles; dual‑control delete
  • Gmail/Drive/Sheets read‑only ingestion
  • EXIF strip on public copy; PDF/A normalization

Compliance

  • M.G.L. c.66 / 950 CMR 32.00; M.G.L. c.30A; 201 CMR 17.00
  • Municipal Records Retention Schedule; NPDES MS4; WCAG 2.1 AA

Acceptance KPIs

K01 Outfall screenings ≥95%
K02 Illicit connections: closed ≤90 days
K03 Catch basin cleaning ≥25%/yr
K04 BMP maintenance on time ≥90%
