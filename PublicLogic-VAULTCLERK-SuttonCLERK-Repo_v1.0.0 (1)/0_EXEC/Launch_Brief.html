Welcome to the SuttonCLERK™ Pilot — our first living deployment under the Frameworks-as-a-Standard™ model. This pilot aligns Google Workspace, Polimorphic, and PublicLogic™ compliance logic, with PublicInsight™ and CSS integrated for readiness, training, and onboarding.

Open Framework Hub Open Package Folder View in Hub
Start Here
  • Use the Framework Hub as your one-screen control center for Promise, 90-Day Plan, Value, Risk, Stack, KPIs, Compliance, Training, and Checklist.
  • Download the seed from 2_PACKAGE, then unzip into a shared drive; the framework runs directly from that directory.
  • Run Quick Setup (GWS) to apply labels, create the Drive watch-folder, and connect the intake sheet.
  • Use Smoke Test to confirm readiness, then open Binder Demo for an export preview.
What’s Inside
  • Framework Core: 21-folder VAULT doctrine (schema, manifests, SOPs, runtime health checks)
  • Governance & Spec: Pilot spec and circular for Town and Polimorphic reference
  • Quick Setup (GWS): Human-readable guide and JSON labels
  • Training Layer: PublicInsight™ — readiness assessment and clerk micro-labs
  • Sample Data & Binder Demo: Three intakes with compiled binder preview
  • Smoke Test: One-click compliance readiness check
  • Integrity & Licensing: LICENSE · README · SHA256SUMS
Kickoff Goals
  • Week 1: Complete Quick Setup, Smoke Test, and Binder Demo; schedule PublicInsight™ micro-labs.
  • Week 2: Begin live intakes via /intake to validate schema and retention chain.
  • By Day 30: Unified queue, verified schema, binder export, and audit-ready compliance log.
  • CSS finalizes support cadence and grant-alignment strategy.

Let’s launch. This becomes our baseline reference build once Sutton is live.
